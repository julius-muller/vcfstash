## Developer sign-off (optional, recommended)

If you are able to do so, please sign off commits to indicate agreement with the
statement above:

```bash
git commit -s -m "Your message"
````

This adds a `Signed-off-by:` line to the commit message.

## How to contribute

1. Fork the repository and create a feature branch.
2. Make your changes with tests and documentation updates where applicable.
3. Ensure tests pass.
4. Open a pull request describing the change and rationale.

## Reporting issues

Please use GitHub Issues and include:

* vcfcache version
* OS / environment
* minimal reproduction steps
* relevant logs and inputs (sanitized if needed)
